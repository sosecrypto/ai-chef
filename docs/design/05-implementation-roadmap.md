# 구현 로드맵

> AI 셰프 앱 개발 단계별 계획

## 기술 스택

| 영역 | 기술 |
|------|------|
| **모바일 앱** | Flutter 3.10+, Riverpod, Go Router |
| **웹 앱** | Next.js 16, React 19, TypeScript, Tailwind CSS |
| **백엔드** | Supabase (PostgreSQL, Auth, RLS, Storage) |
| **AI** | Google Gemini 2.5 Flash/Pro |
| **배포** | Google Cloud Run, Vercel |

## 마일스톤

| 단계 | 목표 | 핵심 기능 |
|------|------|----------|
| **MVP** | 기본 사용 가능 | 인증, 냉장고 관리, 프로필, 기본 레시피 추천 |
| **Alpha** | AI 통합 | 셰프 대화, 재료 기반 레시피, 대체 재료 |
| **Beta** | 고급 기능 | 사진 인식, 알림, 히스토리 |
| **Release** | 완성 | 최적화, 소셜 기능 |

## 우선순위 원칙

1. **핵심 가치 먼저**: "있는 재료로 레시피 추천" 기능 우선
2. **사용성 중심**: 냉장고 관리가 쉬워야 함
3. **개인화 강조**: 셰프 캐릭터로 차별화
4. **보안 필수**: RLS로 데이터 격리 철저히

## 기술 결정 사항

### Flutter 상태 관리: Riverpod
- 타입 안전성
- 테스트 용이성
- 의존성 주입 지원

### 라우팅: Go Router
- 선언적 라우팅
- 딥링크 지원
- 리다이렉트 로직

### API 통신: Supabase Client
- RLS 자동 적용
- 실시간 구독 지원
- 타입 안전한 쿼리

### AI 모델 선택
- **Gemini Flash**: 일반 대화, 빠른 추천
- **Gemini Pro**: 복잡한 레시피 생성, 영양 분석
